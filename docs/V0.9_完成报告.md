## 三、功能交付

### 3.1 Task 023｜MeilisearchStore 适配

**需求背景**：V0.8 仅支持 LocalFileStore 本地文件存储，无法支持生产环境的全文检索和高效查询。需引入 Meilisearch 作为可切换的存储后端。

**交付物**：
- `src/agents/meilisearch-store.ts`：实现 OpportunityStore 接口，同步方法（add/update/delete/get/list）兼容 LocalFileStore 行为，异步方法（search/by_type/by_region）暴露 Meilisearch 唯一能力
- `src/agents/store-factory.ts`：根据 `OPPORTUNITY_STORE_TYPE` 环境变量切换 LocalFileStore/MeilisearchStore，支持 mockMode 回退
- `scripts/migrate-local-to-meili.ts`：一键迁移本地 JSON 数据到 Meilisearch 索引
- `src/api/context.ts`：注入点切换为 createStore

**验收状态**：✅ 98/0 PASS（commit `8ddb965`）

### 3.2 Task 024｜Watch Rules 搜索集成 + T9 增量标签

**需求背景**：搜索结果入库前需经 Watch Rules 过滤，避免无关信息入库；T9 增量标签实现内容 hash 去重，相同内容不重复分析。

**交付物**：
- `src/search/incremental-tagger.ts`：T9 增量标签管理（hashContent SHA-256 + computeChangeRatio + IncrementalTagger 类）
- `src/search/search-dedup-store.ts`：SearchDedupStore 接口 + LocalDedupStore 本地文件实现
- `src/watch/search-integration.ts`：scoredOpportunityToCard + filterByWatchRules + integrateSearchWithWatchRules 端到端管道
- `scripts/verify-task024.ts`：40 项验收测试

**验收状态**：✅ 40/0 PASS（commit `f369d1f`）

### 3.3 Task 025｜Web UI 编辑器 + T8 HTML 交互

**需求背景**：命令行 + 文本文件的用户体验不足，需提供 Web UI 编辑器，支持 DSL 语法高亮、快捷键、规则测试面板。

**交付物**：
- `src/api/routes/web-ui.ts`：静态文件服务（fs.readFileSync + c.body() 兼容方案）
- `web/index.html`：主页面（暗色主题 + 4 Tab：编辑器/测试/预览/帮助）
- `web/styles.css`：暗色/浅色主题（CSS 变量 + 响应式）
- `web/watch-rules-editor.js`：编辑器核心（DSLParser + 7 色语法高亮 + 快捷键 + API 封装）
- `src/api/app.ts`：注册根路径路由

**验收状态**：✅ 26/0 PASS（commit `b942499`）

### 3.4 Task 026｜多 Provider 扩展 + 雷达路由

**需求背景**：V0.8 仅 Serper 单一搜索源，无法覆盖 OPC 政策、文创类雷达需求。需扩展多 Provider 并按雷达类型路由。

**交付物**：
- `src/search/providers/bocha.ts`：博查 Provider（opc_policy/cultural_heritage，reliability B）
- `src/search/providers/exa.ts`：Exa Provider（ai_competition，neural/keyword 模式，reliability B）
- `src/search/providers/google-cse.ts`：Google CSE Provider（opc_policy，gov.cn 限定，reliability A）
- `src/search/radar-router.ts`：雷达路由规则 + URL 去重（normalizeUrl）
- `src/search/provider-registry.ts`：注册 3 个新 Provider
- `src/search/orchestrator.ts`：合并结果时去重
- `scripts/verify-task019d.ts`：修复"无可用 provider"测试

**验收状态**：✅ 39/0 PASS（commit `97aca4b`）
