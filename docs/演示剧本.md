# ChancePing 3 分钟演示剧本

**版本**：V1.1 单雷达最小闭环版
**适用场景**：参赛展示 / 投资人路演 / 内部 Demo
**演示模式**：Demo Mode（DATA_MODE=mock + LLM_MODE=mock，无需 API Key）

---

```bash
# 1. 启动 Demo Mode 服务器（一键启动，无需任何 API Key）
npm run demo

# 2. 打开浏览器（推荐 Chrome，全屏 F11）
#    演示地址：http://localhost:3000/?demo=true
#    注意：?demo=true 参数会显示顶部"Demo 模式"标识

# 3. 字体大小：建议 Ctrl + + 放大到 125%，方便观众观看
```

**演示前检查清单**：
- [ ] 服务器已启动（终端显示"演示地址：http://localhost:3000"）
- [ ] 浏览器打开 `http://localhost:3000/?demo=true`
- [ ] 顶部显示橙色"Demo 模式"标识
- [ ] 首页显示 Logo + 输入框 + 3 个快捷示例
- [ ] 字体放大到适合观众观看

## 时间线（7 个时间点，总时长 3 分钟）

### 0:00 — 痛点引入

**操作**：打开首页 `http://localhost:3000/?demo=true`

**预期效果**：
- 顶部导航栏显示 Logo + "ChancePing 盯机会" + 橙色"Demo 模式"标识
- 首页居中显示大 Logo + 标题 + 副标题
- 副标题："告诉 AI 你要盯什么，它在合适的时间把对的机会推送给你"
- 下方显示输入框 + "开始"按钮
- 3 个快捷示例按钮：AI 比赛 / 政策申报 / 文创项目

**解说词**：
> 每天有无数比赛、政策、项目机会，但信息不等于情报。ChancePing 帮你从信息洪流中筛出真正适合你的机会，在合适的时间提醒你。

---

### 0:30 — 输入需求

**操作**：
1. 在首页输入框输入"我想找适合个人开发者的 AI 比赛"
2. 点击"开始"按钮（或按 Enter）

**预期效果**：
- 自动切换到"需求确认"Tab
- 左侧对话区显示用户消息 + AI 正在输入（typing 动画）
- AI 返回初步理解（Mock 响应，瞬时返回）
- 右侧确认卡显示 7 维度确认度初始值

**解说词**：
> 不需要写复杂的关键词，用自然语言描述你的需求，AI 会理解并追问。比如"我想找适合个人开发者的 AI 比赛"，AI 会理解你的意图并追问细节。

---

### 1:00 — 系统追问

**操作**：
1. 在需求确认页的输入框输入补充信息："奖金 10 万以上，全国范围，个人参赛"
2. 点击"发送"按钮
3. 如有需要，再补充 1-2 轮（如"AI 应用类，有商业计划书"）

**预期效果**：
- AI 追问 7 个维度（雷达类型 / 地区 / 截止日期 / 奖金 / 参赛资格 / 材料要求 / 风险）
- 右侧确认度总分从初始值逐步上升
- 7 维度进度条逐项填充
- "已确认"列表逐步增加项
- "待确认"列表逐步减少项
- 确认度达到 ≥ 90% 时，"开始搜索"按钮变为可点击

**解说词**：
> AI 通过多轮对话确认你的需求，7 个维度逐项填充：雷达类型、地区、截止日期、奖金、参赛资格、材料要求、风险偏好。确认度达到 90% 才生成需求确认卡，避免模糊需求导致搜索结果不准。

---

### 1:30 — 生成雷达规格

**操作**：点击右侧确认卡的"开始搜索"按钮（确认度 ≥ 90% 后启用）

**预期效果**：
- 自动切换到"搜索"Tab
- 顶部状态栏显示"搜索中..."（loading 动画）
- Mock 模式下约 1-2 秒返回结果

**解说词**：
> 需求确认完成后，系统生成雷达规格，自动执行搜索。Demo 模式使用预设的 5 条 Mock 数据，瞬时返回；真实模式会调用搜索源 + LLM 筛选。

---

### 2:00 — 输出机会卡片

**操作**：等待搜索完成

**预期效果**：
- 搜索 Tab 显示 5 条机会卡片
- 每条卡片包含：
  - 等级徽章（S 强烈推荐 / A 推荐 / B 可关注 / C 备选）
  - 标题 + 主办方 + 地区
  - 截止日期（突出显示）
  - 五维评分进度条（匹配度 / 意图 / 证据 / 紧迫度 / 行动成本）
  - 匹配理由（一句话说明为什么推荐）
  - Star 收藏按钮 + 反馈按钮

**解说词**：
> 每条机会有 S/A/B/C 等级，五维评分（匹配度/意图/证据/紧迫度/行动成本），还有匹配理由。S 级是强烈推荐，C 级可关注。评分基于雷达规格与机会信息的匹配度计算。

---

### 2:30 — Star 保存 + 截止提醒

**操作**：
1. 点击某条机会卡片的 Star 按钮收藏（推荐选 S 级机会）
2. 切换到"机会库"Tab（顶部导航栏第 4 个）

**预期效果**：
- Star 按钮变为已收藏状态（金色）
- Toast 提示"已收藏"
- 机会库 Tab 自动加载：
  - 顶部统计概览（总数 / 收藏 / 即将截止 / 已过期）
  - 截止提醒区显示分级提醒（urgent 红 / soon 橙 / warning 蓝）
  - 筛选栏（雷达 / 等级 / 状态 / 仅看收藏 / 排序）
  - 机会列表（显示已收藏的机会）
  - 分页控件

**解说词**：
> 收藏后机会进入机会库，系统自动计算截止日期，分级提醒你哪些机会即将截止：urgent（3 天内）红色、soon（7 天内）橙色、warning（14 天内）蓝色。过期机会自动标记为已过期。

---

### 3:00 — 导出报告 + 收尾

**操作**：
1. 切换到"报告"Tab（顶部导航栏第 5 个）
2. 选择雷达类型（AI 赛事）+ 周期（最近 7 天）
3. 点击"生成报告"按钮
4. 等待预览区显示 Markdown 报告
5. 点击"Markdown"导出按钮（或 HTML / PDF）

**预期效果**：
- 报告预览区显示 Markdown 渲染后的报告
- 报告包含：标题 + 统计概览（总数 / S/A/B/C 分布）+ 机会列表
- 浏览器自动下载报告文件（chanceping-report-*.md）

**解说词**：
> 一键生成雷达报告并导出，分享给团队或存档。报告包含统计概览和机会列表，支持 Markdown / HTML / PDF 三种格式。

**Slogan（收尾）**：
> Tell AI what to watch. ChancePing pings you at the right time.
> 告诉 AI 你要盯什么，ChancePing 在合适的时间提醒你。

## 常见问题应对

**Q：演示时网络断了怎么办？**
A：Demo Mode 不依赖网络，所有数据在本地内存中，可稳定运行。

**Q：演示时浏览器卡了怎么办？**
A：刷新页面（F5），Mock 数据无状态，立即恢复初始状态。

**Q：观众问"真实数据长什么样"？**
A：回答"真实模式会调用搜索源 API + LLM 筛选，数据是实时的；Demo Mode 用预设数据保证可复现"。

**Q：观众问"评分怎么算的"？**
A：回答"五维评分基于雷达规格与机会信息的匹配度，由 LLM 计算；Demo Mode 用预设分数"。

**Q：观众问"提醒怎么推送"？**
A：回答"V1.1 只做页面内提醒；V1.5 会接入微信/邮件/Webhook 推送"。
